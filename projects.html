SpaceSaving Algorithm in a Distributed Setting using Apache Storm

Presentation Slides

The aim of this semester project for the Special Topics in Database Systems graduate course was to study the Spacesaving algorithm for finding the frequent items in data streams and modify it appropriately so as to work in a decentralized environment. Moreover, we had to understand how the Storm platform works and processes the data so as to design an efficient topology for our problem. Therefore, we divide our problem into 4 stages, as it may be seen from the topology chosen. At first stage, we have the spouts which are responsible for reading the tuples coming from the stream. Next, come the wordnormalizer bolts which split the tuples readen to items, all of which are sent to the appropriate bolt of the next processing stage. The main concept is that different instances of the same item, go to the same bolt which is responsible for tracking its frequency. In the third stage of the processing, we have the SpaceSaving Bolts which are responsible for keeping m counters with the functionality described in Spacesaving algorithm and in the last stage we have the FreqItemEstimator Bolts which are responsible for finding which of the items, whose frequency is stored in the m counters, are indeed frequent according to the condition described in the algorithm. In parallel, in order to be able to examine the accuracy of the decentralized algorithm, we implement the CountItems Bolts, which keep track of the real frequency of each item and the FrequentItems Bolts, which based on the real frequencies of each item and the condition that an item must satisfy so as to be characterized as frequent, give the final frequent items. Finally, since we know the true frequent items, we use the precision and recall metric so as to test our algorithm.

Sentimental Analysis on Tweets

Presentation Slides

The aim of this semester project for the Statistical Modelling & Pattern Recognition course was to build a binary classifier for Tweets, which were classified either as negative or positive. The model used was that of bag-of-words and before building the feature vectors for each class some text processing techniques were used so as to remove parts of speech that do not provide useful information. Furthermore, in order to select the most representative features, in our case unigrams, for each class two metrics were taken into account. At first, terms that are frequent in both classes are discarded and secondly we choose the most frequent terms of the class according to tf metric. The classifier used was Multinomial Naive Bayes and some experiments were conducted so as to see how the thresholds chosen for the metrics described above affect the performance of the classifier.
